🧬 Biomedical Waste Segregation Game

An interactive, browser-based educational game designed to improve biomedical waste segregation awareness among healthcare professionals, students, and support staff.

This game transforms mandatory compliance training into an engaging, measurable learning experience, helping healthcare facilities reduce segregation errors, improve regulatory compliance, and strengthen patient and staff safety.

🎯 Purpose

Improper biomedical waste segregation is one of the most common compliance failures in healthcare facilities.
This game aims to:

Reinforce correct waste segregation practices

Reduce human error through repetition and feedback

Make training engaging instead of passive

Provide a digital, scalable alternative to classroom-only training

🕹️ Game Overview

Players are presented with various waste items commonly generated in healthcare settings and must correctly place them into the appropriate biomedical waste category within a time limit.

Core Mechanics

Multiple difficulty levels (Easy / Medium / Hard)

Countdown-based gameplay

Real-time feedback on correct and incorrect segregation

Score tracking and performance summary at the end of each session

🗂️ Waste Categories Covered

The game is aligned with standard biomedical waste segregation rules:

🟡 Yellow – Infectious & pathological waste

🔴 Red – Contaminated recyclable waste

⚪ White (Translucent) – Sharps

🔵 Blue – Glassware & metallic implants

(Categories can be extended or customized as per local regulations.)

📊 Learning Outcomes

By playing the game, users will:

Identify correct disposal methods for common biomedical waste

Understand consequences of improper segregation

Improve recall through timed decision-making

Build confidence in day-to-day waste handling

🧠 Target Audience

Nurses and clinicians

Hospital housekeeping & support staff

Medical & nursing students

Infection control teams

Hospital training & quality departments

🛠️ Tech Stack

HTML5 – Structure

CSS3 – Responsive UI & styling

JavaScript (Vanilla) – Game logic & interactions

Firebase (optional) – Score storage & analytics

Google Analytics (GA4) – Usage and engagement tracking

No backend server is required to run the core game.
